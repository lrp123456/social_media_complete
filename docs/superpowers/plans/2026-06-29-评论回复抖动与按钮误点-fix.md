# 评论回复抖动与按钮误点修复 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 修复两个抖音评论回复 bug —— (1) `scrollCommentArea('top'/'bottom')` 在已到边界时仍执行完整滚动动画造成 9.7 秒抖动；(2) level=1 回复因 `:nth-child(N)` 选择器失效 + `elementFromPoint` 视口外返回 null，点到相邻错误评论。

**Architecture:** Bug 1 在 `scrollCommentArea` 的 for 循环前加前置边界检查（复用已有 `HumanActions.cdpGetDocumentScrollState`），已在边界时直接 `return true` 不触发 `cdpSmartScroll`。Bug 2 把"找根评论容器"和"在该容器内找回复按钮"合并进 `findRootCommentByUsernameContent` 的同一次 `page.evaluate`/`safeEvaluate`，用容器 DOM 引用直接定位回复按钮（消除 `:nth-child` 选择器与 `elementFromPoint` 二次定位），`scrollExpandAndFindTarget` level=1 路径改为直接消费返回的 `replyBtn`，仅当 `replyBtn` 为 null 时才 `scrollRootIntoView` 后按内容重新匹配重试。

**Tech Stack:** TypeScript、tsx 运行时、Jest (ts-jest)、Puppeteer Page、CDP（`HumanActions` 收口层）、Docker Compose（`sm-ts-api` 容器）。

        function findReplyBtnIn(container: Element, vh: number): { x: number; y: number } | null {
          var opsAreas = container.querySelectorAll('[class*="operations-"]');
          for (var oi = 0; oi < opsAreas.length; oi++) {
            var items = opsAreas[oi].querySelectorAll('[class*="item-"]');
            for (var ri = 0; ri < items.length; ri++) {
              if ((items[ri].textContent || '').trim() === '回复') {
                var r = (items[ri] as HTMLElement).getBoundingClientRect();
                if (r.width > 0 && r.height > 0 && r.top < vh) {
                  return { x: Math.round(r.left + r.width / 2), y: Math.round(r.top + r.height / 2) };
                }
              }
            }
          }
          return null;
        }

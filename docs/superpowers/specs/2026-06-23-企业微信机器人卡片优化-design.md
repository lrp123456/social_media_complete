# 企业微信机器人卡片优化设计文档

## 概述

本文档描述企业微信机器人卡片的两项优化：
1. 登录卡片点击跳转逻辑修改
2. AI生成回复从文本消息改为结构化卡片消息

## 目标

1. **登录卡片**：点击后跳转到图片原始链接（方便放大查看），无图片时不跳转
2. **AI回复**：点击"AI生成回复"后，先发送加载状态消息，生成完成后发送结构化结果卡片，支持复制和直接发送

## 技术背景

### 当前技术栈
- 使用 `@wecom/aibot-node-sdk` 的 WebSocket 长连接通道
- 不使用 HTTP API，因此不支持 `response_code` 机制
- `jump_list` type=3 触发的是文本消息，不是 `template_card_event`

### SDK 限制
- `updateTemplateCard()` 必须在收到 `template_card_event` 后 **5 秒内**调用
- AI 生成耗时 3-30 秒，远超 5 秒窗口
- 当前卡片使用 `text_notice` 类型，不支持 `template_card_event`

### 解决方案
采用**"新消息替换"模式**：
- 用户点击按钮 → 机器人收到文本消息
- 立即 reply 一条加载状态消息
- 异步调用 LLM 生成回复
- 主动 sendMessage 推送完成状态卡片

## 设计方案

### 1. 登录卡片跳转修改

#### 当前行为
- `card_action.url` 固定为 `https://work.weixin.qq.com`
- 点击卡片始终跳转到企业微信网页平台

#### 目标行为
- 有图片时：跳转到图片的OSS原始链接
- 无图片时：不可点击（`card_action.type` 设为 `0`）

#### 实现细节
- 修改 `apps/ts-api-gateway/src/services/wechatBotService.ts` 中的 `sendLoginAlert` 方法
- 将 `card_action` 动态设置：
  - 有图片：`{ type: 1, url: imageUrl }`
  - 无图片：`{ type: 0 }`

#### 代码位置
- 文件：`apps/ts-api-gateway/src/services/wechatBotService.ts`
- 方法：`sendLoginAlert`（约第411-476行）
- 关键行：第464行 `card_action: { type: 1, url: 'https://work.weixin.qq.com' }`

### 2. AI回复卡片优化

#### 当前行为
- 点击"AI生成回复"按钮 → 机器人收到文本消息 → 生成AI回复 → 发送文本消息
- 用户需要从文本消息中复制回复内容

#### 目标行为
- 点击"AI生成回复"按钮 → 机器人立即回复加载状态消息 → 异步生成AI回复 → 发送结构化结果卡片
- 用户可以直接在卡片上复制回复或直接发送

#### 交互流程

```
用户点击"🤖 AI 生成回复"（jump_list type=3）
    ↓
机器人收到文本消息 "ai生成 douyin commentCID"
    ↓
① 查询 DB 获取评论信息（5-20ms）
   - 获取评论内容、视频ID、平台信息
   - 设置 suggestionStatus: 'pending'
    ↓
② 立即 reply 加载状态消息
   "⏳ 正在为评论「xxx」生成AI回复，请稍候..."
    ↓
③ 异步调用 LLM 生成回复（耗时 3-30 秒）
   - 使用 .then().catch() 模式，确保异常被捕获
   - 添加 pendingAIGenerations 记录
    ↓
④ 主动 sendMessage 推送完成状态卡片（或失败状态消息）
   - 清理 pendingAIGenerations 记录
```

**异步代码模式要求**：
```typescript
// 必须使用 .catch() 捕获异常，避免 unhandled rejection 导致进程崩溃
replyGenerator.generateReply(ctx)
  .then(result => {
    // 发送结果卡片
    sendResultCard(userid, commentCid, result);
  })
  .catch(err => {
    // 发送失败消息
    sendErrorMessage(userid, commentCid, err.message);
  })
  .finally(() => {
    // 清理 pendingAIGenerations
    pendingAIGenerations.delete(commentCid);
  });
```

#### 消息模板设计

**加载状态消息**（reply）：
```
⏳ 正在为评论「{评论内容}」生成AI回复...
视频ID: {videoId}
平台: {平台}
预计等待: 10-30秒
```

**完成状态卡片**（sendMessage）：
```typescript
{
  card_type: 'text_notice',
  source: { desc: '🤖 AI 回复助手 · 最终结果', desc_color: 0 },
  main_title: {
    title: '✅ AI 回复已生成',
    desc: '回复视频id:{videoId}的评论',
  },
  emphasis_content: {
    title: truncatedReply,  // AI生成的回复，截断至40字符 + "..."
    desc: 'AI 生成',
  },
  sub_title_text: '评论：' + truncatedComment,  // 评论内容，截断至80字符
  horizontal_content_list: [
    { keyname: '平台', value: '抖音' },
    { keyname: '视频ID', value: '{videoId}' },
    { keyname: '评论', value: truncatedComment },  // 截断至80字符
    { keyname: '模型', value: '{model}' },
  ],
  jump_list: [
    { type: 3, title: '📋 复制完整回复', question: '复制回复 {platform} {commentCid}' },
    { type: 3, title: '📤 直接发送', question: 'ai发送 {platform} {commentCid}' },
  ],
  card_action: { type: 1, url: pinfo.cardActionUrl || 'https://creator.douyin.com/creator-micro/interactive/comment' },
}
```

**内容截断规则**：
- `emphasis_content.title`：AI 回复截断至 40 字符，超过部分用 `...` 结尾
- `sub_title_text`：评论内容截断至 80 字符
- `horizontal_content_list` 的 `value`：截断至 80 字符
- 完整内容通过"📋 复制完整回复"按钮获取

**失败状态消息**（sendMessage）：
```
❌ AI 回复生成失败
评论：{评论内容}
错误：{error message}
请稍后重试，或手动回复
```

#### 用户操作

1. **复制回复**：
   - 用户点击"📋 复制回复"按钮
   - 机器人收到文本消息 "复制回复 douyin commentCid"
   - **处理逻辑**：
     1. 根据 `commentCid` 查询评论的 `suggestedReply`
     2. 如果 `suggestionStatus !== 'ready'` 或 `suggestedReply` 为空，返回错误提示
     3. 将回复内容格式化发送，并附加提示"你可以修改后直接发送给机器人"
   - 机器人发送回复文本供用户复制

2. **直接发送**：
   - 用户点击"📤 直接发送"按钮
   - 机器人收到文本消息 "ai发送 douyin commentCid"
   - 机器人直接发送AI回复到评论区

3. **修改后发送**：
   - 用户复制回复内容
   - 修改后发送给机器人
   - 机器人处理发送任务

#### 状态管理

需要添加 `pendingAIGenerations` Map 追踪 AI 生成任务：

```typescript
private pendingAIGenerations = new Map<string, {
  commentCid: string;
  platform: string;
  userid: string;
  startTime: number;
  timeout: NodeJS.Timeout;
}>();
```

## 测试场景

1. **登录卡片跳转测试**：
   - 有图片时：点击卡片应跳转到图片链接
   - 无图片时：点击卡片无反应（不可点击）

2. **AI回复流程测试**：
   - 点击"AI生成回复"按钮，应立即收到加载状态消息
   - AI生成完成后，应收到结构化结果卡片
   - 点击"复制回复"应收到回复文本
   - 点击"直接发送"应发送AI回复到评论区

3. **边界情况测试**：
   - AI生成失败：应收到失败状态消息
   - 重复点击：应返回"正在生成中"提示
   - 生成超时（5分钟）：应返回超时提示

## 错误处理矩阵

| 场景 | 处理方式 |
|------|----------|
| AI生成成功 | 发送完成状态卡片 |
| AI生成失败 | 发送失败状态消息 |
| 生成超时（5分钟） | 发送超时提示，清理 pendingAIGenerations |
| 重复点击同一评论 | 返回"正在生成中，请稍候"提示 |
| 用户点击"直接发送"但回复已被修改 | 使用数据库中的最新回复内容 |

## 风险与注意事项

1. **消息顺序**：
   - 加载状态消息和结果卡片可能因网络原因顺序颠倒
   - **缓解措施**：结果卡片的 `source.desc` 中注明"最终结果"标识

2. **消息数量**：
   - 每次AI生成会发送2条消息（加载状态 + 结果卡片）
   - 可能增加聊天窗口消息数量

3. **并发处理**：
   - 同一用户对同一评论多次点击需要防重复
   - 使用 `pendingAIGenerations` Map 追踪状态
   - 重复点击时返回"正在生成中，请稍候"提示

4. **内容截断**：
   - 企业微信模板卡片字段有长度限制
   - AI 回复和评论内容需要截断适配
   - 完整内容通过"复制回复"按钮获取

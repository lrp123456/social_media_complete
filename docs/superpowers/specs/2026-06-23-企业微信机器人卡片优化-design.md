# 企业微信机器人卡片优化设计文档

## 概述

本文档描述企业微信机器人卡片的两项优化：
1. 登录卡片点击跳转逻辑修改
2. AI生成回复卡片从对话式改为卡片内更新式

## 目标

1. **登录卡片**：点击后跳转到图片原始链接（方便放大查看），无图片时不跳转
2. **AI回复卡片**：点击“AI生成回复”后，卡片内更新显示加载状态，生成完成后显示回复内容，支持复制和直接发送

## 设计方案

### 1. 登录卡片跳转修改

#### 当前行为
- `card_action.url` 固定为 `https://work.weixin.qq.com`
- 点击卡片始终跳转到企业微信网页平台

#### 目标行为
- 有图片时：跳转到图片的OSS原始链接
- 无图片时：不跳转（或跳转到默认管理页面）

#### 实现细节
- 修改 `apps/ts-api-gateway/src/services/wechatBotService.ts` 中的 `sendLoginAlert` 方法
- 将 `card_action.url` 动态设置为 `imageUrl`（如果存在）
- 如果没有图片，设置为空字符串或默认URL

#### 代码位置
- 文件：`apps/ts-api-gateway/src/services/wechatBotService.ts`
- 方法：`sendLoginAlert`（约第411-476行）
- 关键行：第464行 `card_action: { type: 1, url: 'https://work.weixin.qq.com' }`

### 2. AI回复卡片更新

#### 当前行为
- 点击“AI生成回复”按钮 → 机器人收到消息 → 生成AI回复 → 发送文本消息
- 用户需要从文本消息中复制回复内容

#### 目标行为
- 点击“AI生成回复”按钮 → 卡片更新为加载状态 → 生成完成后卡片更新为显示回复内容
- 用户可以直接在卡片上复制回复或直接发送

#### 技术可行性
- 企业微信应用消息支持更新模板卡片（`update_template_card`接口）
- 当前项目使用企微智能机器人SDK（`@wecom/aibot-node-sdk`），走应用消息通道
- 群机器人Webhook不支持更新，但应用消息支持

#### 交互流程

1. **用户收到评论更新提醒卡片**
   - 包含评论内容、视频信息、操作按钮

2. **点击“AI生成回复”按钮**
   - 机器人收到 `ai生成 <platform> <commentCid>` 消息
   - 更新卡片为加载状态（使用 `update_template_card`）
   - 开始生成AI回复

3. **AI生成完成**
   - 再次更新卡片，显示回复内容
   - 添加“复制回复”和“直接发送”按钮

4. **用户操作**
   - **复制回复**：机器人发送回复文本供用户复制
   - **直接发送**：机器人直接发送AI回复到评论区
   - **修改后发送**：用户复制回复内容，修改后发送给机器人

#### 技术实现要点

1. **保存 `response_code`**：
   - 发送卡片时保存返回的 `response_code`
   - 用户点击按钮时，回调事件也会携带 `response_code`
   - 每个 `response_code` 只能使用一次，有效期72小时

2. **更新卡片接口**：
   - 端点：`POST https://qyapi.weixin.qq.com/cgi-bin/message/update_template_card?access_token=ACCESS_TOKEN`
   - 需要 `userids`、`agentid`、`response_code`、`template_card`

3. **状态管理**：
   - 需要记录哪些卡片正在生成AI回复
   - 需要处理生成失败的情况

#### 代码修改点

1. **`apps/ts-api-gateway/src/services/wechatBotService.ts`**：
   - 添加 `updateTemplateCard` 方法
   - 修改 `onMessage` 处理器，处理AI生成回复逻辑
   - 添加 `response_code` 管理

2. **`apps/ts-api-gateway/src/services/monitorService.ts`**：
   - 修改卡片模板，添加AI回复显示区域

3. **`apps/ts-api-gateway/src/services/llmService.ts`**：
   - 可能需要调整AI生成逻辑，支持异步回调

## 测试场景

1. **登录卡片跳转测试**：
   - 有图片时：点击卡片应跳转到图片链接
   - 无图片时：点击卡片不应跳转

2. **AI回复卡片测试**：
   - 点击“AI生成回复”按钮，卡片应更新为加载状态
   - AI生成完成后，卡片应更新为显示回复内容
   - 点击“复制回复”应发送回复文本
   - 点击“直接发送”应发送AI回复到评论区

## 风险与注意事项

1. **`response_code` 管理**：
   - 每个 `response_code` 只能使用一次
   - 需要妥善保存，避免丢失
   - 72小时过期后需要重新发送卡片

2. **错误处理**：
   - AI生成失败时，需要更新卡片显示错误状态
   - 更新卡片失败时，需要回退到文本消息

3. **性能考虑**：
   - 卡片更新可能有延迟
   - 需要处理并发情况

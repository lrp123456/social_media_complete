# 抖音 0s 假成功 / 系统设置 #310 / 快手登录链路 Design

> 日期：2026-06-30
> 范围：抖音复用标签页 0s 假成功、系统设置页 React #310、快手登录检测（HAR）+ 四平台同标签页登录链路
> 关联前序计划：`docs/superpowers/plans/2026-06-30-登录态检测与恢复按钮修复.md`（已完成 detectKuaishouLogin/runDouyinCheck 入口等）

## 背景与根因

### Bug 1 — 抖音复用标签页 0s 假成功
执行 `mon_1782810350508_6` 耗时 0s 显示成功，实际未执行。复现规律：**关闭抖音标签页后监控正常，复用标签页则 0s**。

根因：`navigateToCreatorHome`（`douyinCrawler.ts:198`）的跳过条件 `currentUrl.includes('creator.douyin.com')` 与 `runDouyinCheck` 入口的 `onWorkPage`（要求 `/creator-micro/home` 或 `/creator-micro/content/manage`）不一致。复用标签页时 page 停在上一轮退出策略残留子页（如 `/creator-micro/data/following/follower`，仍在 douyin 域）→ `navigateToCreatorHome` 判"已在 douyin 域"跳过导航 → `runDouyinCheck` 的 `dyOk` 不命中 → fast-fail `return` 正常 `MonitorResult` → `handleJob` 记 success 0s。

关闭标签页后 `bm.connect` 新建 page 是 `about:blank`（不在 douyin 域）→ `navigateToCreatorHome` 走 goto 成功 → 正常监控。

前序 Task 3 只改了 `runDouyinCheck` 入口判导航，未改 `navigateToCreatorHome` 内部跳过条件，两者漂移。

### Bug 2 — 系统设置页 React #310
控制台报 `Minified React error #310`（Rendered more hooks than during the previous render）。

根因：`MaterialTab.tsx:111-115` 三个 early return（`configQuery.isLoading` / `isError` / `!form`）之后才调用 `useMaterialDiskUsage()`，hook 数量在渲染间变化。

### Bug 3 — 快手登录链路三连问题
1. **about:blank 长等待**：手动触发快手更新后浏览器长时间停在 about:blank。`navigateToHome`（`kuaishouCrawler.ts:526`）在 `about:blank` 上执行 `resolveAndClick`（10s 超时）才回退 `page.goto`。
2. **点不到"立即登录"**：未登录分支直接 `sendLoginQR` → `ensureLoginTab` 新建登录标签页，不在当前 cp 页点"立即登录"。
3. **截图串台 / 标签页重复**：`captureQR` 在登录标签页 hostname 校验失败返回 null → fallback 到监控 page 全页截图（串台）；`find()` 依赖 localStorage 标记，丢失就 `openLoginTab` 反复建新页、旧页不清理（重复标签页）。
4. **登录判定脆弱**：仅 `.user-info-dpd` DOM 可见性，无 API 校验，快手改版即失效。

### HAR 判定依据（用户提供）
快手 `cp.kuaishou.com` 页面初始化时调用 `POST https://cp.kuaishou.com/rest/v2/creator/pc/authority/account/current`：
- 登录：`result === 1`、`data.logined === true`、`data.userId` 为正数、`data.userName` 非空
- 未登录：`result === 109`，含 `loginUrl`（指向 `id.kuaishou.com/pass` 或 `passport.kuaishou.com`），或 `data.logined === false`
- HTTP 403/429/超时为风控/异常，**不得判未登录**，须抛 `UNKNOWN_EXCEPTION`

## 设计

### 第 1 节：抖音 0s 假成功修复

1. **`navigateToCreatorHome` 跳过条件对齐**：改为"是否在工作页"（`/creator-micro/home` 或 `/creator-micro/content/manage`），与 `runDouyinCheck` 入口一致。不在工作页就强制导航（点击 → goto 回退），不再因"在 douyin 域"跳过。
2. **fast-fail 改抛错**：`runDouyinCheck` 与 `runKuaishouCheck` 入口导航后仍偏离工作页时 `throw new Error('[平台] 入口导航后仍偏离工作页: ' + url)`，不再 `return` 正常 `MonitorResult`。`executeMonitorCheck` catch 执行退出策略并向上抛 → `handleJob` 标 failed → BullMQ 重试。
3. **抽取共享判定** `isOnDouyinWorkPage(url)`，避免两处判导航条件漂移。

**验收**：复用标签页残留子页被强制导航回 home；导航失败抛错标 failed 重试；日志出现真实 Phase1 elapsed，不再 0s success。

### 第 2 节：系统设置 #310 修复

1. `MaterialTab.tsx` 把 `useMaterialDiskUsage()` 移到组件顶部所有 hook 之后、early return 之前，无条件调用。
2. 自审其余 tab（`GeneralTab`/`CreationTab`/`LlmTab`/`MatrixTab`）hook 调用均在 early return 之前，同类条件 hook 一并修正。

**验收**：进入"系统设置中心 → 素材更新"tab 不再抛 #310，磁盘使用量正常显示。

### 第 3 节：快手登录检测 HAR + 四平台同标签页登录链路

#### A. 快手登录态检测 `detectKuaishouLoginV2(page)`（废弃 `.user-info-dpd` 单一 DOM）

1. **主方案 · 网络层拦截**：复用 `RequestInterceptor`（`registerListener` 机制）。导航到 cp 页后注册对 `https://cp.kuaishou.com/rest/v2/creator/pc/authority/account/current` 的响应拦截，等待页面加载自然触发：
   - `result === 1 && data.logined === true` → 已登录
   - `result === 109` 或含 `loginUrl` 或 `data.logined === false` → 未登录
   - HTTP 403/429/超时/未捕获响应 → `UNKNOWN_EXCEPTION`（抛错，不判未登录）
2. **兜底 · DOM 智能轮询**：N 秒内未拦截到接口时，500ms 步长、8s 上限轮询：
   - 已登录强特征：`ul.el-menu` 存在且含"数据中心/内容管理"等汉字菜单 + `.user__name` 非空
   - 未登录强特征：`div.passport-login-container` 弹窗 或 含 `passport.kuaishou.com` 的 iframe，或"去上传"等元素 href 指向 `/rest/infra/logout`
   - 8s 内都不命中 → 抛 `PAGE_RENDER_TIMEOUT`
3. **`runKuaishouCheck` Phase0** 改用 `detectKuaishouLoginV2`。未登录才进登录流程（C）；异常抛错标 failed 重试。
4. 旧 `detectKuaishouLogin`（`.user-info-dpd`）标记 deprecated，仅供"继续监控"过渡，不再用于监控 Phase0。

#### B. about:blank 长等待修复

`navigateToHome`：`currentUrl` 是 `about:blank` 或非 cp 域时**直接 `page.goto(CREATOR_HOME)`**，跳过空白页 `resolveAndClick` 10s 超时。点击式导航仅用于"已在 cp 域但不在首页"防风控场景。

#### C. 四平台同标签页登录链路（不新建标签页）

**统一原则**：监控 page 就是登录 page，全程不新建标签页 → 根除标签页重复与截图串台（截图永远是当前 page）。

**新增 `performLoginOnCurrentTab(page, userId, platform, flowId)` 取代 `sendLoginQR` 调 `ensureLoginTab` 的路径**：
- **抖音**：URL 不变，页面直接是登录页 → 当前页 `activatePlatformQR` + `captureQR` 截图发企微
- **快手**：URL 不变（cp 页），检测未登录后**在当前页点击"立即登录"**（定位未登录弹窗登录按钮）→ 跳 passport 登录页（URL 变）→ `div.platform-switch` 切 QR → 截图发企微
- **小红书/视频号**：未登录时 URL 已自动变登录页 → 当前页 `captureQR` 截图发企微

**截图统一**：`captureQR(page, config)` 用当前 page；hostname 校验失败说明流程错，**抛错而非 fallback 发错截图**。移除 `sendLoginQR` 中所有 `captureAndSendQR(page, ...)` fallback（串台来源）。

**发卡后状态机**：
- 标 `login_required` → `getAllActiveUsers` 排除出调度（计时器暂停）
- 发一张企微卡片（"恢复登录"/"强制刷新"/"F5刷新"跳转）
- 不阻塞等扫码、不 QR 过期重发循环（移除 `handleLogin` 的 5 分钟 while + maxQrRefreshes）

**"继续监控"指令**（`wechatBotService.ts:711`）同步：在 record.page 导航 + `detectKuaishouLoginV2` 判定，命中 → `markLoginRecovered` + `resetSchedulerTimer`。

#### D. 恢复登录按钮（沿用已实现）

后端 `recoverLogin` + `/recover-login` 路由（直接置 active + 启动倒计时）前序已实现，前端按钮已改。本节不再改，仅验证与新链路兼容：恢复后下一轮监控用 `detectKuaishouLoginV2` 复检，未登录则发卡片（正确行为）。

## 一、页面架构概述

### 1.1 技术架构

视频号助手页面采用腾讯「无界」(wujie) 微前端框架。评论管理页面的 UI 运行在**闭合 Shadow DOM** 内（`wujie-app[data-wujie-id="interaction"]`），外层 document 仅包含左侧导航栏。

**核心穿透方式：**

| 工具 | 穿透方式 | 说明 |
|------|---------|------|
| Playwright | `page.locator()` 默认穿透 open/closed Shadow DOM | 推荐方案，自动穿透 |
| Playwright evaluate | `document.querySelector('wujie-app').shadowRoot` → **null**（closed） | evaluate 内必须手动获取 shadowRoot，但 closed 模式无法获取 |
| Playwright evaluate | 通过 `page.frames()` 找到 `/micro/content/` 的帧 | 备选方案，在帧内执行 JS |
| Chrome Extension | `read_page` / `find` 通过无障碍树部分穿透 | 可穿透侧边栏，但 wujie-app 内部元素不可见 |

**Playwright 是最推荐的自动化工具**，因为它的 locator 机制天然支持 Shadow DOM 穿透。

#### 策略 A：滚动遍历所有视频，筛选评论数 > 0 的视频

```python
async def find_videos_needing_reply(page):
    """在左侧视频列表中查找有评论的视频"""
    container_selector = '.feeds-container'
    videos_with_comments = []
    
    # 1. 先滚动到列表顶部
    await scroll_to_top(page, container_selector)
    
    # 2. 循环滚动，采集所有视频信息
    prev_count = 0
    stable_retries = 0
    MAX_RETRIES = 3
    
    while stable_retries < MAX_RETRIES:
        # 获取当前所有可见视频项的信息
        current_videos = await page.evaluate("""(() => {
            const shadowRoot = document.querySelector('wujie-app').shadowRoot;
            const feeds = shadowRoot.querySelectorAll('.comment-feed-wrap');
            return Array.from(feeds).map(f => ({
                title: f.querySelector('.feed-title')?.textContent?.trim() || '',
                commentCount: parseInt(f.querySelector('.opr')?.textContent?.trim() || '0'),
                rect: f.getBoundingClientRect()
            }));
        })()""")
        
        # 记录评论数 > 0 的视频
        for v in current_videos:
            if v['commentCount'] > 0 and v['title'] not in [x['title'] for x in videos_with_comments]:
                videos_with_comments.append(v)
        
        # 向下滚动加载更多
        await scroll_in_container(page, container_selector, delta_y=300)
        await page.wait_for_timeout(1500)
        
        # 检测是否到底
        new_count = len(current_videos)
        if new_count == prev_count:
            stable_retries += 1
        else:
            stable_retries = 0
        prev_count = new_count
    
    return videos_with_comments
```

### 2.5 滚动机制关键注意事项

**只有真实鼠标滚轮事件能触发懒加载！** 这是最关键的技术难点。

| 方式 | Playwright API | 是否有效 |
|------|---------------|---------|
| 真实鼠标滚轮 | `page.mouse.wheel(0, deltaY)` | ✅ **推荐** |
| `el.scrollTo()` | `page.evaluate(() => el.scrollTo(...))` | ⚠️ 可能有效但不稳定 |
| 直接设 scrollTop | `page.evaluate(() => el.scrollTop = val)` | ❌ 无效 |
| scrollTop + dispatchEvent | `page.evaluate(...)` | ❌ 无效 |

**正确的滚动代码：**

```python
async def scroll_in_container(page, selector, delta_y=300):
    """在指定容器内滚动（能触发 IntersectionObserver 懒加载）"""
    # 1. 获取容器的可视区域中心坐标
    rect = await page.evaluate("""((sel) => {
        const shadowRoot = document.querySelector('wujie-app').shadowRoot;
        const el = shadowRoot.querySelector(sel);
        const r = el.getBoundingClientRect();
        return { x: r.x + r.width / 2, y: r.y + r.height / 2 };
    })(""", selector)
    
    # 2. 移动鼠标到容器中心
    await page.mouse.move(rect['x'], rect['y'])
    
    # 3. 模拟鼠标滚轮
    await page.mouse.wheel(0, delta_y)
```

### 3.4 判断评论是否需要回复的策略

```python
async def get_comments_needing_reply(page):
    """获取当前视频中需要回复的评论"""
    comments = await page.evaluate("""(() => {
        const shadowRoot = document.querySelector('wujie-app').shadowRoot;
        const commentWrap = shadowRoot.querySelector('.feed-comment__wrp');
        const innerDiv = commentWrap.querySelector('.scroll-list > div');
        const rootItems = innerDiv.querySelectorAll(':scope > .comment-item');
        
        const results = [];
        rootItems.forEach((item, index) => {
            const main = item.querySelector('.comment-item-main');
            if (!main) return;
            
            const username = main.querySelector('.comment-user-name')?.textContent?.trim() || '';
            const isAuthor = !!main.querySelector('.comment-author-bandage');
            const content = main.querySelector('.comment-content')?.textContent?.trim() || '';
            const time = main.querySelector('.comment-time')?.textContent?.trim() || '';
            
            // 检查是否有子回复（作者回复）
            const replyList = main.querySelector('.comment-reply-list');
            const hasAuthorReply = replyList && 
                Array.from(replyList.querySelectorAll('.comment-author-bandage')).length > 0;
            
            // 检查是否有"展开更多回复"按钮
            const loadMoreBtn = main.querySelector('.load-more__btn');
            const hasMoreReplies = loadMoreBtn && 
                loadMoreBtn.textContent.includes('展开更多回复');
            
            // 获取评论操作按钮的位置（用于后续点击回复）
            const actions = main.querySelector('.comment-actions');
            const actionRect = actions ? actions.getBoundingClientRect() : null;
            
            results.push({
                index,
                username,
                isAuthor,
                content,
                time,
                hasAuthorReply,    // 作者是否已回复
                hasMoreReplies,    // 是否有更多未展开的回复
                actionRect,        // 操作按钮位置
                needsReply: !isAuthor && !hasAuthorReply  // ★ 非作者自己发的 + 作者尚未回复
            });
        });
        
        return results;
    })()""")
    
    # 筛选需要回复的评论
    return [c for c in comments if c.get('needsReply', False)]
```

**判断逻辑：**

1. **跳过作者自己的评论**：检查 `.comment-author-bandage` 是否存在，如果评论者是作者本人，则不需要回复
2. **检查是否已有作者回复**：在 `.comment-reply-list` 中查找是否有带 `.comment-author-bandage` 标签的子回复
3. **优先回复未回复的评论**：`needsReply = !isAuthor && !hasAuthorReply`

### 4.2 查找并点击回复按钮

```python
async def click_reply_button(page, comment_index):
    """
    点击指定评论的回复按钮
    
    comment_index: 评论在列表中的索引（0-based）
    """
    # 1. 获取目标评论的回复按钮位置
    btn_info = await page.evaluate("""((commentIndex) => {
        const shadowRoot = document.querySelector('wujie-app').shadowRoot;
        const commentWrap = shadowRoot.querySelector('.feed-comment__wrp');
        const innerDiv = commentWrap.querySelector('.scroll-list > div');
        const rootItems = innerDiv.querySelectorAll(':scope > .comment-item');
        
        const targetItem = rootItems[commentIndex];
        if (!targetItem) return null;
        
        const actions = targetItem.querySelector('.comment-actions');
        if (!actions) return null;
        
        // 回复按钮是 .comment-actions 中的第二个可点击元素
        // 图标按钮通常为 SVG 图标包裹在 button/div/span 中
        const actionBtns = actions.querySelectorAll('button, [role="button"], .action-btn, span[class*="action"]');
        
        // 策略1: 通过按钮数量索引，回复通常是第2个
        if (actionBtns.length >= 2) {
            const replyBtn = actionBtns[1];
            const r = replyBtn.getBoundingClientRect();
```

## 六、元素定位策略总结

### 6.1 最可靠的定位方式（按优先级排序）

1. **CSS 类名选择器**（仅限 wujie Shadow DOM 内的稳定类名）：
   - `.comment-feed-wrap` → 视频项
   - `.feed-title` → 视频标题
   - `.comment-item` → 评论项
   - `.comment-content` → 评论文本
   - `.comment-actions` → 操作按钮区
   - `.comment-author-bandage` → 作者标签
   - `.load-more__btn` → 展开更多回复按钮

2. **文本匹配**（对于有明确文字的按钮/标签）：
   - "全部评论" → 评论面板标题
   - "共 N 条评论" → 评论总数
   - "展开更多回复" → 子回复展开按钮
   - "发送" / "取消" → 回复输入区按钮

3. **位置索引**（对于图标按钮，无文字可匹配时）：
   - `.comment-actions` 内第2个图标 → 回复按钮
   - `.comment-actions` 内第1个图标 → 点赞按钮

### 6.2 不可靠的定位方式

1. **动态 hash 类名**：如 `.container-sXKyMs`、`.comment-content-text-JvmAKq` 等带随机后缀的类名，每次构建可能变化
2. **DOM 层级路径**：硬编码的 XPath 路径容易随版本更新失效
3. **元素 ID**：大多数元素没有稳定的 ID

## 九、风险与注意事项

1. **Shadow DOM 穿透**：Playwright 的 `locator` 默认穿透 Shadow DOM，但 `page.evaluate` 中获取 closed shadowRoot 会返回 null。推荐使用 Playwright locator 进行元素查找，或使用 `page.frames()` 切换到子应用帧。

2. **滚动必须用 `mouse.wheel`**：不能用 `page.evaluate` 设置 `scrollTop`，这会跳过 IntersectionObserver 回调。

3. **滚动步长控制**：每次 `wheel` 的 deltaY 建议 200~400px，过大可能跳过哨兵元素触发区域。

4. **回复按钮的悬停行为**：部分操作按钮可能只在鼠标悬停评论行时才显示，需要先 hover 到评论区域再点击。

5. **回复输入框的类型**：可能是 `textarea` 或 `contenteditable` 的 `div`，需要两种都尝试。

6. **提交方式**：优先使用 Enter 回车键提交，备选点击"发送"按钮。

7. **评论数口径**：页面显示的评论总数包含子回复，实际根评论数可能少于显示数。

8. **并发风险**：不要同时操作多个视频的评论，页面状态可能混乱。建议逐个视频串行处理。

9. **频率控制**：API 请求间隔 ≥ 1.5 秒，回复间隔 ≥ 6 秒，避免触发微信反爬机制。

10. **Cookie 保活**：长时运行需处理 Cookie 保活，建议维持专用浏览器窗口。

### 1.1 现有实现概况

| 功能模块 | 抖音 | 快手 | 视频号（参考） |

### 1.2 关键发现

**快手回复功能完全缺失**：`monitorService.ts` 的 `executeReplyAction()` 函数中仅有 `douyin` 和 `tencent` 的处理分支，没有 `kuaishou` 分支。同时 `wechatBotService.ts` 中回复任务的 `platform` 字段硬编码为 `'douyin'`，不支持快手平台。

`selectors.json` 中快手部分也缺少回复按钮、回复输入框、发送按钮等关键选择器配置。

### 2.3 回复评论完整操作流程 `[浏览器验证]`

#### 前置条件

- 已登录抖音创作者中心
- 浏览器页面位于 `creator.douyin.com` 域名下
- 已进入评论管理页面并加载了评论列表

#### 步骤 1：导航到评论管理页面

- **操作**：导航至 `https://creator.douyin.com/creator-micro/interactive/comment`
- **或**：点击左侧菜单 `#douyin-creator-master-menu-nav-comment_manage_new`
- **等待时间**：2000-3000ms（页面加载及评论数据请求）
- **验证**：页面标题显示"评论管理"，评论列表已渲染

#### 步骤 2：定位目标评论

- **操作**：遍历 `div.container-sXKyMs` 元素，通过 `div.comment-content-text-JvmAKq` 的文本内容匹配目标评论
- **选择器**：`[class*="container-sXKyMs"]`
- **注意**：当前版本 **没有** `data-cid` 属性，无法通过评论 ID 直接定位，需要通过文本匹配或 API 拦截获取映射关系

#### 步骤 3：点击回复按钮

- **目标元素**：在目标评论的 `div.operations-WFV7Am` 容器中，找到文本为"回复"的 `div.item-M3fSkJ`
- **CSS 选择器**：`[class*="operations"] [class*="item-"]` 中 `textContent.trim() === '回复'` 的元素
- **备选选择器**：`text=回复`（全局文本匹配，需限定在当前评论容器内）
- **触发方式**：**始终可见**，不需要 hover 即可看到回复按钮
- **点击后等待**：500-1000ms（输入框展开动画）

#### 步骤 4：输入回复内容

- **输入框位置**：点击回复按钮后，在该评论下方展开回复区域
- **回复区域容器**：`div.reply-content-obcMk0`
- **输入框选择器**：`div.reply-content-obcMk0 div[contenteditable="true"]`
- **输入框 class**：`input-d24X73`
- **Placeholder**：`"回复 {用户名}："`（如 "回复 lqq："）
- **输入方式**：逐字输入（CDP keyPress），50-150ms/字
- **字数限制**：待验证

> **注意**：页面上有两个 `contenteditable="true"` 输入框：一个是顶部的主评论输入框（placeholder: "有爱评论，说点好听的~"），另一个是点击回复后出现的回复输入框（placeholder 含"回复"字样）。操作时需选择正确的输入框。

#### 步骤 5：发送回复

- **发送按钮选择器**：`button.douyin-creator-interactive-button-primary:not([class*="disabled"])` 中 `textContent === '发送'` 的按钮
- **完整 HTML**：
  ```html
  <button class="douyin-creator-interactive-button
                 douyin-creator-interactive-button-primary
                 douyin-creator-interactive-button-size-small">
    <span class="douyin-creator-interactive-button-content">发送</span>
  </button>
  ```
- **空内容时状态**：按钮带 `douyin-creator-interactive-button-primary-disabled` class，`aria-disabled="true"`，**不可点击**
- **有内容时状态**：按钮 class 变为 `douyin-creator-interactive-button-primary`，**可点击**
- **取消按钮**：`button.douyin-creator-interactive-button-tertiary` + 文本"取消"，用于关闭回复输入框
- **点击后等待**：2000-3000ms（网络请求 + 页面更新）

#### 步骤 6：验证发送结果

- **成功标识**：
  - 页面出现 Toast 提示：**"回复成功"**（class: `douyin-creator-interactive-toast`）
  - 回复输入框自动关闭
  - 新回复出现在目标评论下方，包裹在 `div.reply-list-QwXCb_` 容器中
  - 新回复带有红色 **"作者"** 标识（`span.douyin-creator-master-badge`）
- **失败标识**：待验证（未触发过失败场景）

#### 评论列表 API（代码推断 + 抓包确认）

| 接口 | 方法 | 路径 | 说明 |

### 2.6 页面技术栈特征 `[浏览器验证]`

| 技术项 | 详情 |
|--------|------|

| 图标系统 | `dux-icon` + SVG 内联图标 |
| 类型系统 | TypeScript |

**选择器稳定性提示**：CSS Modules 哈希后缀（如 `container-sXKyMs` 中的 `-sXKyMs`）在每次构建时可能变化。建议优先使用以下策略：

1. **文本内容匹配**：`text=回复`、`text=发送` — 最稳定
2. **Semi Design 组件 class**：`douyin-creator-interactive-button-primary` — 较稳定
3. **属性选择器**：`[contenteditable="true"]` — 稳定
4. **哈希 class 的模糊匹配**：`[class*="container-"]`、`[class*="operations"]` — 中等稳定性（前缀不变，后缀可能变）

### 2.7 现有代码实现参考 `[代码推断]`

```typescript
// 文件：monitorService.ts (executeReplyAction 函数)
// 抖音平台回复流程

try {
  // 1. 导航到创作者中心 & 评论管理
  await douyinCrawler.navigateToCreatorHome(page);
  await douyinCrawler.navigateToCommentManage(page);

  // 2. 打开作品选择抽屉 & 选择视频
  await douyinCrawler.openSelectWorkDrawer(page);
  await douyinCrawler.findAndClickVideoInDrawer(page, replyData.videoId, '');

  // 3. 定位目标评论（通过容器选择器 + 文本匹配）
  const containers = await HumanActions.queryElementsWithInfo(
    page, '[class*="container-sXKyMs"]'
  );

  // 4. 点击"回复"按钮
  await HumanActions.cdpClickByText(page, '回复');

  // 5. 聚焦输入框 & 逐字输入
  await HumanActions.cdpClick(page, 'div[contenteditable="true"]');
  for (const char of replyData.text) {
    await HumanActions.cdpKeyPress(page, char, char, char.charCodeAt(0));
    await HumanActions.wait(page, 50, 150);
  }

  // 6. 点击发送
  await HumanActions.cdpClick(page, '[class*="submit"]');
} catch (error) {
  logger.error('抖音回复失败:', error);
  throw error;
} finally {
  await douyinCrawler.executeExitStrategy(page);
}
```

> **需要更新的地方**：
> 1. 发送按钮选择器 `[class*="submit"]` 可能不再准确，实际页面使用的是 `button.douyin-creator-interactive-button-primary`
> 2. 定位评论的 `[class*="container-sXKyMs"]` 仍然有效，但需要配合文本匹配而非 `data-cid`
> 3. 建议输入框选择器限定为 `[class*="reply-content"] div[contenteditable="true"]`，避免误操作顶部主输入框

### 3.4 回复评论完整操作流程 `[浏览器验证]`

#### 前置条件

- 已登录快手创作者中心
- 浏览器页面位于 `cp.kuaishou.com` 域名下
- 已进入评论管理页面（`/article/comment`）并加载了评论列表

#### 步骤 1：导航到评论管理页面

- **操作**：导航至 `https://cp.kuaishou.com/article/comment`
- **或**：点击左侧菜单中的"评论管理"菜单项
- **等待时间**：2000-3000ms（页面加载及评论数据请求）
- **验证**：页面标题显示"评论管理"，评论列表已渲染

#### 步骤 2：定位目标评论

- **操作**：遍历 `div.comment-item` 元素，通过 `div.comment-content__detail` 的文本内容匹配目标评论
- **选择器**：`.comment-item`
- **注意**：没有 `data-cid` 等属性存储评论 ID，需通过文本匹配或 API 拦截获取映射关系

#### 步骤 3：点击回复按钮

- **目标元素**：在目标评论的 `div.comment-content__btns` 中，找到包含 `div.btn-icon.icon-reply` 的 `div.comment-content__btns__btn`
- **CSS 选择器**：`.comment-item .comment-content__btns .icon-reply` 的父元素 `.comment-content__btns__btn`
- **备选选择器**：通过按钮 icon 的 class `icon-reply` 定位
- **触发方式**：**始终可见**，不需要 hover 即可看到回复按钮图标
- **点击后等待**：500-1000ms（输入框显示，`display` 从 `none` 变为 `block`）
- **⚠️ 重要**：JavaScript 的 `.click()` 和 `dispatchEvent(MouseEvent)` **无法触发** 回复输入框显示！必须通过 Vue 组件方法或真实的浏览器交互（如 CDP/坐标点击）触发

> **技术细节**：快手使用 Vue.js 2 框架，事件绑定在 Vue 组件层面。直接调用 DOM 的 `.click()` 不会触发 Vue 的事件处理器。需要：
> 1. 通过 `element.__vue__` 获取 Vue 组件实例，调用其 `onClickReply` 方法
> 2. 或使用真实的浏览器鼠标事件（CDP `Input.dispatchMouseEvent`）
> 3. 或使用 Playwright/Puppeteer 的 `click()` 方法（会触发完整事件链）

#### 步骤 4：输入回复内容

- **输入框位置**：点击回复按钮后，在该评论的操作栏下方展开回复区域

- **输入框 class**：`comment-input`
- **Placeholder**：`"回复 {用户名}："`（如 `"回复 User_1508319381044："`）
- **输入方式**：逐字输入（CDP keyPress），50-150ms/字

- 创作者中心 API（`/rest/cp/creator/` 路径）的签名要求相对宽松，主要依赖 Cookie 认证

```typescript
// 伪代码 - 推荐的快手实现模式
async function executeKuaishouReplyAction(task, replyData) {
  try {
    // 1. 导航到评论管理页面（注意：URL 是 /article/comment，不是 /comment）
    await page.goto('https://cp.kuaishou.com/article/comment');
    await page.waitForSelector('.comment-item', { timeout: 5000 });

    // 2. 定位目标评论（通过文本匹配）
    const comments = await page.$$('.comment-item');
    let targetComment = null;
    for (const comment of comments) {
      const text = await comment.$eval('.comment-content__detail', el => el.textContent.trim());
      if (text === replyData.commentText) {
        targetComment = comment;
        break;
      }
    }

    // 3. 点击回复按钮（⚠️ 必须用 Playwright click，不能用 JS .click()）
    const replyBtn = await targetComment.$('.icon-reply');
    await replyBtn.click();  // Playwright click 会触发完整事件链
    await page.waitForTimeout(500);

    // 4. 输入回复内容
    const input = await targetComment.$('.comment-input__wrapper .comment-input');
    await input.click();
    for (const char of replyData.text) {
      await page.keyboard.type(char, { delay: 50 + Math.random() * 100 });
    }

    // 5. 点击确认按钮（注意：文本是"确认"不是"发送"）
    const submitBtn = await targetComment.$('.comment-btn.sure-btn');
    await submitBtn.click();
    await page.waitForTimeout(2000);

    // 6. 验证结果（检查新的子回复是否出现）
  } catch (error) {
    logger.error('快手回复失败:', error);
    throw error;
  } finally {
    await kuaishouCrawler.executeExitStrategy(page);
  }
}
```

> **⚠️ 关键技术差异**：快手使用 Vue.js 2，DOM 的 `.click()` 方法不会触发 Vue 事件。必须使用 Playwright/Puppeteer 的 `.click()` 方法，它会通过 CDP 发送真实的鼠标事件，从而触发 Vue 的事件处理器。

### 4.2 实现前注意事项

基于浏览器实际验证，以下是开发快手回复功能时需要特别关注的事项：

1. **页面 URL 是 `/article/comment`**，不是 `/comment`（后者重定向到 `/profile`）
2. **无需选择视频**：评论管理页面直接展示所有视频的评论，不需要先选择视频再操作
3. **发送按钮文本是"确认"**：选择器不能用 `text=发送`
4. **Vue.js 事件兼容性**：必须使用浏览器自动化框架的 click 方法，不能用 JavaScript `.click()`
5. **`__NS_sig3` 签名**：如果直接调用 API 需要此签名，建议通过 UI 自动化规避
6. **评论 ID 获取**：DOM 中没有评论 ID 属性，需要通过 API 拦截（`commentList` 接口响应）获取评论 ID 映射

## 六、模拟人类行为的关键要求

### 6.1 鼠标移动

- 不要瞬间移动到目标位置，需要有移动轨迹
- 移动过程中加入轻微的随机偏移
- 到达目标后等待 200-500ms 再点击

### 6.2 键盘输入

- 逐字输入，每字间隔 50-150ms
- 偶尔加入更长的停顿（模拟思考）
- 不要使用粘贴方式一次性输入

### 6.3 点击行为

- 点击前先将鼠标移动到目标元素上方
- 等待 200-500ms 后执行点击
- 点击后等待元素响应（300-500ms）

### 6.4 操作间隔

- 每个主要操作之间等待 500-2000ms
- 加入随机的额外等待时间
- 避免过于规律的操作节奏

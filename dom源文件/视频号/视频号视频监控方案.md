# 视频号评论监控方案 — 技术设计文档

> 版本：v1.0 | 日期：2026-06-10
> 平台：微信视频号助手 (channels.weixin.qq.com)
> 对标：抖音评论监控方案 (douyinCrawler.ts)
> 状态：方案设计阶段，部分API需登录后DevTools二次验证

---

## 一、平台概况与技术特征

### 1.1 平台身份

视频号助手是微信视频号官方创作者管理后台，域名为 `channels.weixin.qq.com`。与抖音创作者中心、快手创作者中心定位相同，是评论监控的数据源入口。

### 1.2 核心技术架构

| 属性 | 值 | 监控影响 |
|------|-----|---------|
| 应用名 | `finder-helper-web` | 日志/UA标识 |
| 前端框架 | Vue 3 SPA | DOM渲染时序 |
| 微前端 | **wujie (无界)** — 腾讯自研 | 子应用DOM隔离，需特殊处理 |
| 组件库 | WeUI设计体系 | CSS选择器前缀 |
| API前缀 | `/cgi-bin/mmfinderassistant-bin/` | API拦截模式 |
| 内容模块API | `/micro/content/cgi-bin/mmfinderassistant-bin/` | 视频/评论API路径 |
| 登录方式 | **仅微信扫码** (无账号密码) | 登录自动化受限 |
| CSS前缀 | `finder-ui-desktop-menu__*` | 菜单选择器 |
| JS钩子 | `js_nav_item` | 高稳定性导航选择器 |
| Vue Scoped | `data-v-2a16615a` (侧边栏) | 低稳定性，避免使用 |

### 1.3 wujie 微前端架构

视频号平台与抖音/快手/小红书最大的技术差异在于使用了 wujie (无界) 微前端框架：

```
主壳应用 (finder-helper-web)
├── 侧边栏 (#side-bar, #menuBar)         ← 主应用直接渲染
├── 路由管理 (History API)                ← 主应用控制
├── 登录/鉴权                             ← 主应用控制
│
└── 子应用 (<wujie-app data-wujie-id="xxx">)
    ├── content  → 内容管理/互动管理模块   ← /micro/content/*
    ├── live     → 直播模块               ← /micro/live/*
    └── eccommerce → 电商/带货模块        ← /micro/eccommerce/*
```

**对监控的影响**：
- 视频列表、评论列表等核心DOM在 `<wujie-app>` 子应用内，不在主文档DOM树中
- 通过 `document.querySelector('wujie-app')` 可访问子应用容器
- API请求路径带 `/micro/content/` 前缀，这是子应用模块标识
- CDP层面的DOM查询不受wujie隔离影响（CDP直接操作渲染树）

### 1.4 与抖音方案的关键差异

| 维度 | 视频号 | 抖音 |
|------|--------|------|
| 登录 | 微信扫码 (需人工介入) | Cookie注入 |
| 微前端 | wujie无界 (子应用DOM隔离) | 无 |
| 视频列表API | `POST post/post_list` | `GET /web/api/creator/item/list` |
| 评论数来源 | 视频列表API直接包含 | 需单独获取 |
| API签名 | Session Cookie (较简单) | msToken+a_bogus (复杂) |
| 菜单DOM | `finder-ui-desktop-menu__*` | `douyin-creator-*` |
| 推荐指标 | 有 (recommend_count, 视频号特有) | 无 |
| 数据延迟 | 数据中心有数小时延迟 | 实时 |

## 二、三阶段监控流水线

与抖音方案完全对齐，视频号评论监控同样采用三阶段流水线架构：

```
┌──────────────────────────────────────────────────────────┐
│  Phase 1: 视频列表发现                                    │
│  导航到视频管理页 → 拦截 post_list API → 获取视频ID/评论数 │
│  对比 sync_state → 发现新视频 + 检测评论数变化             │
├──────────────────────────────────────────────────────────┤
│  Phase 2: 导航到评论管理                                  │
│  点击侧边栏"互动管理 > 评论" → 等待评论页加载             │
│  选择目标视频 → 拦截评论列表 API                          │
├──────────────────────────────────────────────────────────┤
│  Phase 3: 评论采集与入库                                  │
│  一级评论翻页 → 子回复展开 → 评论入库 → 增量去重           │
│  (可选) 拟人化回复                                        │
└──────────────────────────────────────────────────────────┘
```

## 三、Phase 1：视频列表发现

### 3.1 导航到视频管理页

**入口URL**: `https://channels.weixin.qq.com/platform/post/list`

**导航策略** (按优先级):

1. **菜单点击** (首选，最安全):
   - 点击侧边栏 "内容管理" 展开子菜单
   - 点击子菜单 "视频"
   - 选择器: `getByText('span', '视频', exact=true)` 或 `.finder-ui-desktop-sub-menu__item:has(span:has-text('视频'))`

2. **直接导航** (回退):
   - `page.goto('https://channels.weixin.qq.com/platform/post/list')`
   - 仅在菜单点击失败时使用

### 3.4 增量检测逻辑

```typescript
// 与抖音 checkForUpdates() 完全对齐的逻辑
async checkForUpdates(
  page: Page,
  userId: number
): Promise<CheckResult> {
  // 1. 导航到视频列表页
  await this.navigateToVideoList(page);

  // 2. 注册API监听
  await this.registerVideoListListener(page);

  // 3. 触发数据加载 (刷新或滚动)
  await HumanActions.cdpF5Refresh(page);
  HumanActions.clearCDPContext(page);
  await HumanActions.wait(page, 3000, 5000);

  // 4. 获取拦截到的视频列表
  const intercepted = await this.interceptor.waitForResponse(
    POST_LIST_PATTERN, 15000
  );
  const videos: TencentVideoInfo[] = intercepted?.body?.list || [];

  // 5. 对比 sync_state
  const previousState = await db.getSyncState('tencent', userId);
  const commentsQueue: CommentQueueItem[] = [];

  for (const video of videos.slice(0, this.maxMonitorVideos)) {
    const oldCount = previousState?.lastCommentCounts?.[video.export_id] ?? 0;
    const newCount = video.object_stat?.comment_count ?? 0;

    if (newCount > oldCount) {
      commentsQueue.push({
        exportId: video.export_id,
        description: video.desc,
        oldCount,
        newCount,
        isFirstCrawl: oldCount === 0,
        _userId: userId,
      });
    }
  }

  this.unregisterListener();

  return {
    hasUpdate: commentsQueue.length > 0,
    commentsQueue,
    updatedVideos: videos.map(v => ({
      exportId: v.export_id,
      desc: v.desc,
      oldCount: previousState?.lastCommentCounts?.[v.export_id] ?? 0,
      newCount: v.object_stat?.comment_count ?? 0,
    })),
  };
}
```

## 四、Phase 2：导航到评论管理

### 4.1 评论管理页入口

**入口URL**: `https://channels.weixin.qq.com/platform/comment`

**导航策略** (参考 douyinCrawler.navigateToCommentManage):

```typescript
async navigateToCommentManage(page: Page): Promise<boolean> {
  logger.info('[Phase2] Navigating to comment management page');

  await HumanActions.thinkingPause(page, 800, 2000);
  await this.ensureSidebarReady(page);

  const maxRetries = 3;
  for (let attempt = 0; attempt < maxRetries; attempt++) {
    // 检测是否已在评论页
    const bodyText = await HumanActions.cdpGetBodyText(page);
    const alreadyOnCommentPage = bodyText.includes('评论')
      && (bodyText.includes('切换视频') || bodyText.includes('全部类型'));

    if (alreadyOnCommentPage) {
      logger.info({ attempt }, '[Phase2] Already on comment page');
      return true;
    }

    // 方式1: 通过菜单点击
    // 先展开 "互动管理" 父菜单
    const interactClicked = await resolveAndClick(
      page, 'menu.interact', 'tencent', { timeout: 8000 }
    );
    if (interactClicked) {
      await HumanActions.wait(page, 1000, 2000);
    }

    // 再点击 "评论" 子菜单
    const commentClicked = await resolveAndClick(
      page, 'menu.interact.comment', 'tencent', { timeout: 10000 }
    );

    if (commentClicked) {
      logger.info('[Phase2] Comment menu clicked, waiting for page load');
      await HumanActions.wait(page, 3000, 5000);

      const loaded = await this.waitForCommentManagePage(page);
      if (loaded) {
        logger.info('[Phase2] Comment page loaded');
        return true;
      }
    }

    // 方式2: 直接导航 (回退)
    if (attempt === maxRetries - 1) {
      await page.goto(
        'https://channels.weixin.qq.com/platform/comment',
        { waitUntil: 'domcontentloaded' }
      );
      await HumanActions.wait(page, 3000, 5000);
      const loaded = await this.waitForCommentManagePage(page);
      if (loaded) return true;
    }

    await HumanActions.wait(page, 2000, 3000);
  }

  logger.error('[Phase2] Failed to navigate to comment page');
  return false;
}
```

### 5.2 评论采集流程

```typescript
async processCommentsQueue(
  page: Page,
  queue: CommentQueueItem[],
  userId: number
): Promise<CommentProcessResult[]> {
  const results: CommentProcessResult[] = [];

  // 导航到评论管理页
  const navigated = await this.navigateToCommentManage(page);
  if (!navigated) {
    return queue.map(q => ({
      exportId: q.exportId,
      success: false,
      comments: [],
      error: 'Failed to navigate to comment page',
    }));
  }

  // 注册评论API监听
  await this.registerCommentListener(page);

  for (const item of queue) {
    try {
      // 选择目标视频
      if (item.description) {
        await this.selectVideoForComments(
          page, item.exportId, item.description
        );
      }

      // 等待评论API响应
      const commentResp = await this.interceptor.waitForResponse(
        COMMENT_LIST_PATTERN, 15000
      );

      const comments = commentResp?.body?.list || [];
      const allComments: CommentInfo[] = [];

      // 处理一级评论
      for (const c of comments) {
        allComments.push(this.mapComment(c, item.exportId, 1));

        // 如果有子回复，展开并获取
        if (c.reply_count > 0) {
          const replies = await this.fetchSubReplies(
            page, c.comment_id, item.exportId
          );
          allComments.push(...replies);
        }
      }

      // 入库
      await db.batchUpsertComments('tencent', allComments, userId);

      results.push({
        exportId: item.exportId,
        success: true,
        comments: allComments,
      });

      // 视频间冷却
      await HumanActions.wait(page, 3000, 5000);

    } catch (error: any) {
      logger.error({ error: error.message, exportId: item.exportId },
        '[Phase3] Comment processing failed');
      results.push({
        exportId: item.exportId,
        success: false,
        comments: [],
        error: error.message,
      });
    }
  }

  this.unregisterListener();
  return results;
}
```

## 六、登录与会话管理

### 6.1 登录流程

视频号仅支持微信扫码登录，这是与抖音/快手/小红书最大的差异点。

```
1. 导航到 https://channels.weixin.qq.com/login.html
2. 等待 iframe[src*="login-for-iframe"] 加载完成
3. 截取二维码区域 → 发送给用户/管理后台
4. 轮询检测登录状态 (每2秒):
   ├── URL 包含 "/platform" → 登录成功
   ├── 页面文本 "已扫码需在手机上进行确认" → 等待确认
   └── 页面文本 "二维码已过期" → 刷新二维码重新等待
5. 成功后 Cookie 自动写入 .weixin.qq.com 和 channels.weixin.qq.com
```

**代码实现**:

```typescript
async handleLogin(page: Page, userId: number): Promise<boolean> {
  logger.info('[Login] Navigating to login page');
  await page.goto('https://channels.weixin.qq.com/login.html', {
    waitUntil: 'domcontentloaded',
  });

  await HumanActions.wait(page, 2000, 4000);

  // 截取二维码
  const qrFrame = await page.$('iframe[src*="login-for-iframe"]');
  if (qrFrame) {
    const screenshot = await qrFrame.screenshot();
    // 通过 WebSocket/Redis 推送给管理后台
    await db.saveLoginQRCode(userId, screenshot);
  }

  // 轮询登录状态 (最长等待 120 秒)
  const maxWait = 120000;
  const startTime = Date.now();

  while (Date.now() - startTime < maxWait) {
    const url = page.url();
    if (url.includes('/platform')) {
      logger.info('[Login] Login successful');
      return true;
    }

    const bodyText = await HumanActions.cdpGetBodyText(page);
    if (bodyText.includes('已过期')) {
      logger.warn('[Login] QR code expired, refreshing');
      // 点击刷新
      await HumanActions.cdpClick(page, '.qrcode-refresh-btn', {
        fallbackText: '刷新',
        timeout: 5000,
      });
      await HumanActions.wait(page, 2000, 3000);
      // 重新截图
      const newQrFrame = await page.$('iframe[src*="login-for-iframe"]');
      if (newQrFrame) {
        const newScreenshot = await newQrFrame.screenshot();
        await db.saveLoginQRCode(userId, newScreenshot);
      }
    }

    await HumanActions.wait(page, 2000, 3000);
  }

  logger.error('[Login] Login timeout after 120s');
  return false;
}
```

## 七、拟人化回复

### 7.1 回复操作流程

与抖音回复流程对齐，但操作对象为视频号评论页DOM：

```
1. 定位目标评论的 "回复" 按钮 → 文本匹配 "回复" 或 .reply-btn
2. HumanActions.cdpClick() 点击回复
3. 等待输入框出现 → .reply-textarea 或 textarea[placeholder*="回复"]
4. HumanActions.safeCDPType() 逐字输入 (50-150ms/字)
5. 等待发送按钮可用 (非 disabled 状态) → .submit-reply-btn
6. HumanActions.cdpClick() 点击发送
7. 等待发送成功反馈
8. 冷却 6-12 秒
```

**回复延迟对比**:

| 平台 | 单条延迟 | 输入速度 |
|------|---------|---------|
| 视频号 | 6-12s | 50-150ms/字 |
| 抖音 | 6-12s | 50-150ms/字 |
| 快手 | 3-6s | 50-150ms/字 |
| 小红书 | 10-15s | 50-150ms/字 |

### 7.2 风控检测

```typescript
const RISK_CONTROL_KEYWORDS = [
  'captcha', '验证', '安全', '限制', '封禁',
  '账号异常', '操作频繁', 'login'
];
const RISK_CONTROL_URLS = ['/login', '/verify', '/captcha'];

async detectRiskControl(page: Page): Promise<RiskControlDetection> {
  const url = page.url();

  // URL 检测
  for (const riskUrl of RISK_CONTROL_URLS) {
    if (url.includes(riskUrl) && !url.includes('/platform')) {
      return {
        detected: true,
        type: 'url_redirect',
        evidence: `Redirected to: ${url}`,
      };
    }
  }

  // 页面文本检测
  const bodyText = await HumanActions.cdpGetBodyText(page);
  for (const keyword of RISK_CONTROL_KEYWORDS) {
    if (bodyText.includes(keyword)) {
      return {
        detected: true,
        type: 'risk_keyword',
        evidence: `Found: "${keyword}" in page text`,
      };
    }
  }

  return { detected: false, type: '', evidence: '' };
}
```

### 8.5 monitorService.ts — 添加 tencent 分支

```typescript
// executeMonitorCheck() 中添加:
case 'tencent': {
  const tencentCrawler = new TencentCrawler(config.maxMonitorVideos);

  // 登录检测
  const currentUrl = page.url();
  if (currentUrl.includes('/login')) {
    const loggedIn = await tencentCrawler.handleLogin(page, userId);
    if (!loggedIn) throw new Error('Tencent login failed');
  }

  // Phase 1: 检测更新
  const checkResult = await tencentCrawler.checkForUpdates(page, userId);

  if (checkResult.hasUpdate) {
    // Phase 2+3: 采集评论
    const commentResults = await tencentCrawler.processCommentsQueue(
      page, checkResult.commentsQueue, userId
    );

    // 更新 sync_state
    await db.updateSyncState('tencent', userId, {
      videoIds: checkResult.updatedVideos.map(v => v.exportId),
      lastCommentCounts: Object.fromEntries(
        checkResult.updatedVideos.map(v => [v.exportId, v.newCount])
      ),
    });

    return commentResults;
  }

  return [];
}
```

## 九、选择器策略总结

### 9.1 稳定性分级

| 等级 | 策略 | 示例 | 适用场景 |
|------|------|------|---------|
| S | `#id` | `#menuBar`, `#side-bar` | 侧边栏容器 |
| S | 文本匹配 | `getByText('span', '评论')` | 菜单项/按钮 |
| A | 组件前缀类 | `.finder-ui-desktop-menu__link_current` | 菜单状态 |
| A | JS钩子类 | `.js_nav_item` | 导航项 |
| B | WeUI图标 | `.weui-icon-outlined-home` | 图标定位 |
| C | `ml-key`属性 | `[ml-key="assistant_tab_privateMsg"]` | 特定元素 |
| D | Vue scoped | `[data-v-2a16615a]` | **避免使用** |

### 9.2 菜单导航降级链

```
文本匹配 ("内容管理")
  ↓ 失败
.js_nav_item:has(span:has-text('内容管理'))
  ↓ 失败
.finder-ui-desktop-menu__sub__wrp:has(span:has-text('内容管理'))
  ↓ 失败
直接 page.goto('/platform/post/list')
```

### 9.3 wujie 微前端处理策略

由于视频列表和评论列表DOM在 `<wujie-app>` 子应用内，选择器需要注意：

1. **CDP层面**: `DOM.querySelector` / `DOM.getDocument` 可直接穿透 wujie 容器，无需特殊处理
2. **CSS选择器**: `wujie-app .video-card` 可作为回退选择器
3. **避免**: `document.querySelector('.video-card')` 可能找不到元素（wujie DOM隔离）
4. **推荐**: 优先使用 API 拦截获取数据，DOM 操作仅用于导航和回复

### 11.2 已知风险

1. **微信扫码登录无法完全自动化**: 每次会话过期需人工扫码，建议通过管理后台推送二维码图片
2. **wujie微前端DOM隔离**: 部分CSS选择器可能无法穿透子应用，需以API拦截为主、DOM操作为辅
3. **评论API路径未确认**: 推测的API路径可能不准确，实际实现时需根据Network面板数据修正
4. **会话有效期不确定**: Cookie有效期从数小时到数天不等，需实现健壮的过期检测和重登机制
5. **数据中心数据延迟**: 数据中心的统计数据有数小时延迟，不宜作为增量检测的主要数据源
